# STELLA Benchmark Evaluation Criteria

## 1. Evaluation Dimensions

### 1.1 Technical Accuracy (5.0 points)
Measures the correctness and precision of scientific content and methodology.

Scoring Criteria:
- 5.0: Perfect accuracy with comprehensive detail
- 4.5-4.9: Near-perfect with minor omissions
- 4.0-4.4: Very accurate with some minor errors
- 3.5-3.9: Generally accurate with few significant errors
- 3.0-3.4: Mostly accurate but with notable gaps
- <3.0: Major errors or omissions

Key Assessment Points:
- Data accuracy and reliability
- Methodology correctness
- Calculation precision
- Protocol accuracy
- Database query accuracy
- Tool and version specificity
- Parameter validation
- Error handling consideration

### 1.2 Domain Knowledge (5.0 points)
Evaluates understanding of bioengineering concepts and principles.

Scoring Criteria:
- 5.0: Expert-level understanding with cross-domain integration
- 4.5-4.9: Deep understanding with comprehensive context
- 4.0-4.4: Strong understanding with good context
- 3.5-3.9: Good understanding with some context
- 3.0-3.4: Basic understanding with limited context
- <3.0: Superficial or incorrect understanding

Key Assessment Points:
- Biological concept understanding
- Technical terminology usage
- Cross-disciplinary integration
- Current research awareness
- Field-specific best practices
- Literature citation accuracy
- Protocol standardization knowledge
- Regulatory compliance awareness

### 1.3 Analytical Quality (5.0 points)
Assesses depth and sophistication of analysis.

Scoring Criteria:
- 5.0: Comprehensive multi-level analysis with novel insights
- 4.5-4.9: Deep analysis with strong supporting evidence
- 4.0-4.4: Thorough analysis with good reasoning
- 3.5-3.9: Good analysis with some depth
- 3.0-3.4: Basic analysis with limited depth
- <3.0: Superficial or flawed analysis

Key Assessment Points:
- Analysis depth
- Logical reasoning
- Data interpretation
- Statistical rigor
- Critical evaluation
- Alternative analysis methods
- Result validation approaches
- Limitation discussion
- Edge case consideration

### 1.4 Innovation Impact (5.0 points)
Evaluates creative problem-solving and practical applicability.

Scoring Criteria:
- 5.0: Groundbreaking insights with immediate practical value
- 4.5-4.9: Novel approaches with clear applications
- 4.0-4.4: Creative solutions with good utility
- 3.5-3.9: Useful insights with some creativity
- 3.0-3.4: Standard approaches with limited innovation
- <3.0: Minimal innovation or impractical solutions

Key Assessment Points:
- Solution creativity
- Practical applicability
- Future implications
- Alternative approaches
- Improvement suggestions
- Resource optimization
- Scalability considerations
- Cross-platform compatibility
- Integration potential

### 1.5 Communication Quality (5.0 points)
Assesses clarity, organization, and professionalism of output.

Scoring Criteria:
- 5.0: Exceptional clarity with perfect organization
- 4.5-4.9: Excellent clarity with strong organization
- 4.0-4.4: Very clear with good organization
- 3.5-3.9: Clear with adequate organization
- 3.0-3.4: Somewhat clear with basic organization
- <3.0: Unclear or poorly organized

Key Assessment Points:
- Information structure
- Clarity of expression
- Professional formatting
- Completeness
- Accessibility
- Documentation quality
- Error message clarity
- Visual aid usage
- Reference formatting

## 3. Scoring Methodology

### 3.1 Score Calculation
- Individual dimension scores: Evaluated to 0.25 point precision
- Overall score: Weighted average of dimensions
  * Technical Accuracy: 25%
  * Domain Knowledge: 20%
  * Analytical Quality: 20%
  * Innovation Impact: 15%
  * Communication Quality: 20%

### 3.2 Score Modifiers
- Task completion rate impact
- Error severity adjustments
- Innovation bonus points
- Complexity handling bonus
- Cross-domain integration bonus
- Resource efficiency bonus
- Documentation quality impact
- Reproducibility factor

### 3.3 Quality Control
- Multiple evaluator consensus
- Reference answer comparison
- Industry standard alignment
- Consistency checking
- Regular calibration
- Peer review process
- Version control tracking
- Feedback incorporation
- Continuous improvement

### 3.4 Model-Specific Considerations
- Response consistency
- Resource utilization
- API integration capability
- Error recovery
- Context retention
- Tool usage proficiency
- Documentation generation
- Code quality (when applicable)
- Performance optimization

### 3.5 Evaluation Process
1. Initial Assessment
   - Task understanding verification
   - Resource availability check
   - Prerequisite validation

2. Execution Monitoring
   - Step-by-step validation
   - Error tracking
   - Resource usage monitoring
   - Time efficiency analysis

3. Result Validation
   - Output verification
   - Cross-reference checking
   - Peer review
   - Documentation review

4. Score Assignment
   - Dimension-specific scoring
   - Modifier application
   - Final score calculation
   - Review and adjustment

5. Documentation
   - Score justification
   - Improvement suggestions
   - Reference materials
   - Future considerations
